## [1.1.0] ŌĆö 2026-05-05

References bundle, multi-match router, and template consumption. The MCP server bundles the 45 reference files the Darkforge Claude Code plugin uses; every `forge_*` call routes through that bundle and shapes generation accordingly. forge_dark gains an optional `description` field and the same router consultation.

### Added

- `refs/` ŌĆö bundled reference library (45 markdown files, ~44,000 lines) mirrored from the plugin's `skills/forge/references/`.
- `scripts/sync-refs.mjs` ŌĆö maintainer script that syncs `refs/` from the plugin (local checkout via `--from-local <path>` or GitHub via `--from-github`) and regenerates the routing table.
- `src/lib/reference-loader.ts` ŌĆö loads markdown by logical name (e.g. `01-framer-motion`, `patterns/hero`).
- `src/lib/reference-router.ts` ŌĆö multi-match router with always-loaded `00-dark-tokens`, hand-maintained style/intent synonyms (glass, neon, stagger, partner, marquee, chip, ŌĆ”), and the auto-generated literal-name table. Returns up to 4 references ranked tokens ŌåÆ lib ŌåÆ pattern.
- `src/lib/reference-router.generated.ts` ŌĆö auto-generated routing rows from the plugin's `SKILL.md` Phase 2 table. Regenerated by `npm run sync-refs`.
- `ReferenceExcerpt` type and `REFERENCE_EXCERPT_CHARS` constant in `src/types/index.ts`.
- `get_reference` tool ŌĆö returns the markdown text of any bundled reference.
- `list_references` tool ŌĆö lists every bundled reference grouped by libraries vs patterns.
- Specificity rubric in `scripts/smoke.mjs` (`scoreSpecificity`) ŌĆö regex-based 6-point check used by the smoke harness on the partner-logos cross-check prompts.

### Changed

- `forge_component`, `forge_skeleton`, `forge_page` ŌĆö every call routes through the reference router and threads loaded markdown into template builders. Variant selection, list builders, layout direction, motion stagger, and import set all consult `referenceTexts`. Imports and `prefersReduced` are now derived from the assembled body (no dead imports). `tokensRequired` is regex-derived from the final code, not template-declared. Description-named entities ("for Microsoft, AWS, Google Cloud") are extracted and preserved as item content. When `componentType` and description disagree on layout (`feature-grid` + `strip`), the description wins.
- `forge_dark` ŌĆö optional `description` field added. When provided, the router fires and per-element treatments (glass utilities, color-mixed borders, named glow tokens) get applied based on description hints. Without `description`, behavior is unchanged from v1.0.
- `structuredContent` for routed tools always includes `routedReferences: string[]` and `referenceExcerpts: ReferenceExcerpt[]` (each excerpt sliced to 800 chars).

### Verified

The same partner-logos prompt that scored 0/5 against the v1.1 first cut now scores 6/6 on `forge_component` (glass + glow + prefersReduced + stagger + horizontal layout + entity preservation) and 5/6 on `forge_dark` (everything except motion stagger, which would require schema-changing imports).

### Notes

- References are read-only in this repo. Source-of-truth lives in the plugin (Karannnnn614/Darkforge); sync flows plugin ŌåÆ MCP only.
- Bundle size: ~1 MB markdown, npm-acceptable.
- Tool count: 8. Schema additions only ŌĆö no removes, no renames.

## [1.0.0] ŌĆö 2026-05-03

First release. Six MCP tools for forging AMOLED-dark UI from any MCP-compatible AI tool ŌĆö Claude Code, Cursor, Windsurf, Gemini CLI, Codex CLI, Zed.

### Added

- `scan_stack` ŌĆö reads `package.json`, returns a Darkforge stack profile (animation, components, 3D, styling, framework, language).
- `list_libraries` ŌĆö lists every UI library Darkforge knows about, filterable by category. Marks installed/not-installed when given a project path.
- `forge_component` ŌĆö generates one complete AMOLED-dark TSX/JSX component from a freeform description, tuned to the detected stack. Variant (dark/glass/neon/minimal) inferred from description keywords.
- `forge_skeleton` ŌĆö generates a skeleton loading state matching a real component's structure. Pulse + shimmer variants.
- `forge_dark` ŌĆö converts a light Tailwind/inline component to AMOLED dark with `--df-*` tokens; produces a diff-style change log.
- `forge_page` ŌĆö generates a multi-section dark page (saas/product/portfolio/agency/startup/dashboard) tuned to the detected stack.
- `scripts/smoke.mjs` ŌĆö stdio smoke harness exercising every tool.
- Token system ŌĆö full `--df-*` CSS variable block embedded in `src/lib/token-system.ts`.

### Notes

- Companion to the Darkforge Claude Code plugin (separate repo, same brand, same token system, same design language).
